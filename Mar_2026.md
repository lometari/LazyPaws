**1. GitHub - что делать? (генерация ключей, добавлени репозиториев и т.д.)**
1. Естественно установить Git на пк
2. Генерация ключей
Ключи лежат в по умолчанию в папке С:\Users\Username\.ssh
2.1 Генерация (происходит через командную строку, находясь в папке с ключами)
```bash
ssh-keygen -t ed25519 -C "your_email", 
```
где 
-t ed25519 
Указывает тип ключа
-C "your_email"
добавляет комментарей с заданным эл.адресом, не влияет на шифрование, информационное дополнение.

Нужно будет ввести имя ключа и пароль(опционально). Будет создана пара ключей: приватный и публичный, последний нужно добавить на аккаутне в GitHub.

2.2. Добавление ключа
На GitHub
Скопировать содержимое файла keyName.pub в раздел настроек Settings -> SSH and GPG keys.

В Git Bash:
```bash
eval "$(ssh-agent -s)" //запуск агента
ssh-add ~/.ssh/privat_key_name // добавление приватного ключа
```

2.3. Проверка репо
В моем случае все оказалось просто, нужно было перейти в мой репо LazyPaws ввести команду
```bash
git remote
```
и оказалось, что гит помнит, что это репозиторий и где находится его удаленная копия. Дальше просто:
```bash
git status //проверить изменения. Надо же у нас новый файл Mar_2026.md
git add -A //добавить все файлы
git commit -m "комментарий комита" 
git push
```

2.3.1 Нежданчик
Оказалось нужно внести изменения в имя ключа в файле config по адресу c:/Users/username/.ssh
После этого push сработал и зеленый квадратик засветился в графике на GitHub! 🟩
